# Icon Library — 图标库生成规范

图标是独立设计资产，必须先生成图标文件目录，再由 Components 和 Preview 统一引用。AI 不得在页面或组件中临时画图标。

---

## 一、产物要求

每次生成 UI 设计交付物时，必须同时生成独立 SVG 图标目录。目录位置、文件命名和相对引用路径以 [artifact-structure.md](./artifact-structure.md) 为准。

HTML 设计文档必须通过 `<svg><use ...></use></svg>` 引用这些独立图标文件。

禁止把完整图标内容直接复制进 HTML。HTML 中只允许出现引用图标的 `<svg><use ...></use></svg>`。

---

## 二、SVG 文件结构

每个图标文件都必须是一个独立 SVG 文件，结构如下：

```xml
<svg xmlns="http://www.w3.org/2000/svg">
  <symbol id="icon-home" viewBox="0 0 24 24">
    <path d="..." />
  </symbol>
</svg>
```

### 必须遵守

- 每个图标使用 `<symbol>` 定义
- 每个 symbol 必须有 `id` 和 `viewBox="0 0 24 24"`
- id 必须使用 `icon-` 前缀，例如 `icon-home`
- 文件名必须与 symbol id 对应，例如 `icon-home.svg` 中必须定义 `id="icon-home"`
- 禁止在 symbol 内写 `style` 属性
- 禁止引用外部图片、字体、CDN 或远程资源
- 同一套图标必须保持线性/面性风格一致
- 禁止把文字、数字或业务缩写画成图标主体，即使它们是用 `<path>` 绘制也不允许；例如血氧图标不能画成 `O2`，血压图标不能画成 `BP`

---

## 三、图标范围与隐喻

AI 必须根据产品实际页面生成所需图标。不要为了凑完整图标库生成无关图标。若场景未明确，可从以下通用图标中选择实际会使用的部分：

| id | 用途 |
|----|------|
| `icon-home` | 首页/概览 |
| `icon-device` | 设备/硬件 |
| `icon-report` | 报告/文档 |
| `icon-profile` | 用户/我的 |
| `icon-search` | 搜索 |
| `icon-settings` | 设置 |
| `icon-alert` | 提醒/异常 |
| `icon-plus` | 添加 |
| `icon-check` | 成功/完成 |
| `icon-close` | 关闭/失败 |

业务图标的隐喻应来自用户如何理解这个概念，而不是来自领域模板。生成业务图标前先问：

1. 用户在日常语言里如何称呼这个对象或动作？
2. 这个概念有没有用户熟悉的实物、动作、状态或方向隐喻？
3. 24x24 下是否一眼可辨，还是必须依赖文字标签？
4. 图标是否会与同页其他图标、状态色或数据符号混淆？
5. 是否正在用领域刻板印象替代真实语义，例如把所有健康指标都画成波形或液滴？

业务缩写只能作为文字标签出现，不能作为图标主体。例如 `BP`、`O2`、`HR` 可以作为指标文字，不得放在图标容器中替代图标。

---

## 四、Icon Library 页面

最终交付目录必须包含 `icon-library.html`，作为图标总览页面。

`icon-library.html` 必须展示：

- 图标预览
- 图标名称
- symbol id / 文件名
- 使用场景

展示的图标也必须引用真实图标文件，不得复制 symbol 内容到 HTML。

---

## 五、引用规则

根目录页面和子目录页面的图标引用路径以 [artifact-structure.md](./artifact-structure.md) 为准。

禁止：

- 使用字母、数字、emoji 代替图标
- 使用 `<path>` 绘制字母、数字或业务缩写来伪装成图标
- 在 Preview 中临时写 `<path>`
- Components 和 Preview 使用不同来源的图标
- 使用外部图标库链接
- 在 `components/` 或 `pages/` 中直接复制图标 SVG 内容而不是引用图标文件

---

## 六、质量标准

- 图标语义必须清晰，一眼能看出用途
- 图标复杂度适中，24x24 下仍可辨认
- 同一套图标线宽、端点、转角、视觉重心一致
- 图标不能比文字抢视觉层级
- `icon/` 目录必须只包含最终设计会用到或组件库会展示的图标，避免无意义膨胀
