# Visual Hierarchy — 注意力权重分配规范

UI 的层级不是先规定“几个重点”，而是先理解用户任务，再把有限注意力按比例分配给页面区域。页面设计应像注意力机制一样工作：先计算每个信息单元的重要性，再把权重映射到视觉强度。

---

## 一、注意力分配原则

生成页面前，AI 必须在内部完成一张注意力权重表，不需要展示给用户。

步骤：

1. 拆分页面信息单元：导航、标题区、主要内容、列表项、操作按钮、辅助面板、元信息、状态标签等。
2. 评估每个信息单元对当前用户任务的贡献。
3. 将所有信息单元的注意力权重归一化为 100%。
4. 根据权重决定视觉强度，而不是凭感觉给所有元素加粗、加色块或加阴影。

注意力是相对关系。同一个组件在不同页面可以有不同权重，例如“今日会议”在日历首页可能是高权重，在统计页可能只是低权重明细。

---

## 二、注意力评估因子

为每个信息单元评估以下因子。不同产品、页面和用户状态下，因子优先级会变化；不要套固定公式。

| 因子 | 含义 | 高分示例 |
|------|------|----------|
| 任务相关度 | 是否直接服务当前页面核心任务 | 当前日程、关键健康状态、待处理订单 |
| 决策价值 | 用户是否需要基于它做判断 | 风险提醒、趋势变化、异常数据 |
| 行动可能性 | 用户是否很可能立刻操作 | 添加事件、确认、回复、处理告警 |
| 时间敏感度 | 是否与现在、截止时间、实时状态有关 | 下一场会议、即将超时、当前同步中 |
| 状态严重度 | 是否代表异常、危险、失败或强反馈 | 高风险、支付失败、设备离线 |
| 出现频率 | 是否是常驻框架或重复信息 | 导航、普通标签、历史列表、说明文字 |

评估方式：

- 先判断当前页面的核心任务，再决定哪些因子最重要。
- 若页面强调处理风险，状态严重度和行动可能性权重会上升。
- 若页面强调浏览与选择，任务相关度、决策价值和可扫读性更重要。
- 若页面强调实时监控，时间敏感度会显著影响视觉权重。
- 出现频率高的信息通常应降低噪声，除非它代表当前选中或异常状态。

内部可以用分数帮助思考，但不要把分数或系数当成模板。最终目标是形成相对权重关系，并让这种关系影响视觉。

---

## 三、权重关系与视觉响应

注意力权重是连续关系，不是固定档位。权重越高，越应获得更清晰的位置、更强可读性和更多视觉通道；权重越低，越应依附上下文并降低噪声。

使用方式：

- 最高权重信息应先被看到，但不一定要同时使用大字号、强色彩和强阴影。
- 中等权重信息应清晰可扫读，帮助用户完成比较和判断。
- 低权重信息应保持可读，但不要独立抢视线。
- 权重接近的元素应组织成同一视觉群组，而不是彼此竞争。
- 权重差明显的元素必须呈现可感知的视觉强弱差。

---

## 四、视觉通道映射

权重必须映射到多个视觉通道，而不是只靠颜色。

| 视觉通道 | 高权重做法 | 低权重做法 |
|----------|------------|------------|
| 位置 | 首屏中心、主内容流顶部、与用户视线起点一致 | 侧栏、下方、靠近所属内容 |
| 尺寸 | 更大的容器、更长的可读宽度、更明显的触控区 | 紧凑行高、小标签、从属布局 |
| 字号 | 根据结构和阅读距离提升尺寸 | 小字号可以保留高语义权重，不自动等于低价值 |
| 字重 | 用于局部停顿、当前状态和关键值 | 低权重信息通常较轻，但不是所有小字都必须细 |
| 文字颜色/对比 | 用于主信息、语义状态、可交互性和弱化 | 低权重文字可降低对比，但关键小字仍需足够清晰 |
| 对比 | 明确背景差、清楚边界、高可读文本 | 弱背景、细边框、低饱和文本 |
| 色彩 | 少量主色或状态色强调 | 中性色、低饱和辅助色 |
| 阴影/层级 | 需要浮起时使用轻到中阴影 | 多用边框或背景层级，不加阴影 |
| 留白 | 周围留白更充分，形成视觉停顿 | 与所属信息靠近，避免独立成块 |
| 密度 | 信息更少但更清晰 | 信息可更密，但必须可扫读 |

高权重元素不需要同时用满所有通道。若一个区域已经通过位置和尺寸占优，颜色、字重和阴影应克制。字体的字号、字重、颜色是三个独立旋钮，不能绑定成单调阶梯。

---

## 五、注意力守恒

页面总注意力视为 100%。提高一个区域的视觉权重时，必须降低其他区域的视觉噪声。

- 不要让导航、标签、普通卡片、主按钮同时抢主内容的注意力。
- 主色、粗体、大字号、强阴影、饱和状态色都属于注意力资源。
- 重复列表项应共享一种安静样式；只有当前项、异常项或最相关项获得更高权重。
- 元信息依附于主信息，不应拥有独立卡片级强调。
- 常驻框架默认低权重；只有当前选中项可略微提升。

设计失败的典型表现：每个模块都有彩色背景、每个标题都加粗、每个按钮都像主按钮、每个标签都像可点击操作。

---

## 六、场景校准方法

不要为某个行业预设固定权重表。不同产品、不同页面、不同用户状态下，同一信息单元的注意力权重会变化。AI 应根据页面意图动态校准。

### 校准问题

生成页面前，在内部回答这些问题：

1. 用户打开这个页面最可能要完成什么任务？
2. 哪些信息会改变用户下一步判断？
3. 哪些操作需要立即可见，哪些可以延后出现？
4. 哪些内容只是帮助理解上下文，不应独立抢注意力？
5. 当前状态是否存在异常、临近截止、实时变化或风险？
6. 哪些区域是常驻框架，应该稳定但低噪声？

### 权重倾向

只使用倾向判断，不使用固定百分比模板：

- 任务型页面：把更多注意力给可操作内容和当前流程状态。
- 数据型页面：把更多注意力给关键异常、趋势变化和用户需要判断的指标。
- 浏览型页面：把更多注意力给内容入口、筛选线索和当前上下文。
- 创作型页面：把更多注意力给画布、编辑对象和直接操作工具。
- 监控型页面：把更多注意力给异常、实时状态、优先级和可恢复操作。
- 设置型页面：把更多注意力给当前配置项、风险操作和保存反馈。

### 轻量推理示例

示例只展示思考方式，不是模板：

```text
页面：个人日程首页
用户任务：快速知道接下来要做什么，并能调整今天安排。
高权重：当前时间附近的日程、今日视图范围、添加或调整日程入口。
中权重：当天其他日程、周视图、待办摘要。
低权重：侧边导航、月份切换、历史分类、说明文字。
视觉映射：当前日程用更清晰的位置和对比；普通日程保持可扫读；导航稳定但低噪声。
```

```text
页面：设备监控首页
用户任务：判断设备是否正常，并优先处理异常。
高权重：异常设备、离线状态、恢复或查看详情操作。
中权重：正常设备列表、最近同步时间、关键指标趋势。
低权重：账户入口、历史记录、说明信息。
视觉映射：异常不只靠红色，还通过位置、图标、文字和操作入口共同提升权重。
```

### 动态调整

- 正常状态下，页面应克制，避免把普通数据设计成告警。
- 异常状态下，异常信息和恢复操作可以临时获得更高权重。
- 新手场景中，引导和解释可以提高权重；熟练用户场景中，应降低说明性内容权重。
- 高频工具界面中，导航和框架要稳定但安静；内容和操作效率优先。
- 展示型界面中，品牌、情绪和故事可以获得更高权重，但仍要服务用户目标。

---

## 七、验证方法

生成页面后必须做注意力验证：

1. 权重回读：看页面时能否说出哪几个区域占据最多注意力，以及为什么。
2. 眯眼测试：模糊看页面时，高权重区域是否仍先被看到。
3. 灰度测试：去掉颜色后，字号、位置、留白和容器强度是否仍能表达权重差异。
4. 噪声检查：若低权重元素因为彩色、粗体、阴影或过大尺寸抢眼，必须降噪。
5. 比例检查：相邻区域的重要性不同，视觉强度也必须不同；重要性接近，则应被组织成同一个视觉组。
